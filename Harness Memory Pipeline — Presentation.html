🧠 The Problem

Before

  • Engineering knowledge trapped in Obsidian — invisible during dev sessions
  • Claude Code sessions produce rich work that evaporates after closing
  • No structured way to surface past decisions, gotchas, or project context
  • Weekly sessions follow the same debugging rabbit holes

After

  • Semantic search across all notes — surfaced via MCP inside Claude Code
  • Session summaries auto-written back to Obsidian every week
  • Bidirectional flow: notes → context → work → session notes → indexed
  • Signal-to-noise filtering keeps only what matters

Vector Infrastructure

ingest.py — Obsidian → ChromaDB

Ingest Pipeline

Configuration

  • Embedding model: nomic-embed-text (Ollama)
  • Chunk size: 1,000 chars, overlap: 100
  • Collection: obsidian_notes, HNSW cosine
  • Batch size: 50 chunks per upsert
  • Parser: LangChain ObsidianLoader

Modes

  • Incremental (--incremental): skips files unchanged since last run via .last_ingest timestamp
  • Full re-index (--full): wipes the collection and rebuilds from scratch
  • Metadata: file path + YAML tags attached to every chunk
  • Default outcome=verified_success for MCP filter compatibility

03

MCP Integration

server.py — The Harness

MCP Tools

query_notes(query)

  • Semantic search on ChromaDB
  • Returns top 5 matching chunks
  • Default filter: outcome = verified_success
  • Converts cosine distance to a 0–1 relevance score
  • Results include source path + score

get_project_context(project)

  • Reads Projects/<name>/_Brief.md from vault
  • Case-insensitive project name fallback
  • Lists available projects if not found

🚀 Server runs via FastMCP (Python), registered in ~/.claude/settings.json,
launched at macOS login via a launchd plist — persistent daemon.

Session Sync Pipeline

sync_sessions.py — Closing the Loop

The Gatekeeper

Every session passes through a 3-stage filter before becoming knowledge

1

Heuristic Filter — Structural Check

No tool calls → TRASH  ·  Same Bash command repeated >10x consecutively → TRASH  ·  No git diff changes → RESEARCH

2

Outcome Signal — Success Verification

Non-zero exit code on final hook → flag as failure  ·  Last user message contains positive keywords (that worked, thanks, done, merged) → boost to KNOWLEDGE

3

LLM Classifier — Semantic Check (llama3:8b)

Sends final 20% of transcript to llama3:8b via Ollama. Bug fixes, architectural decisions, non-obvious findings → KNOWLEDGE. Failure, confusion, trivial chat → TRASH.

Obsidian Note Format

One note per calendar day — aggregated across all repos and sessions

---
session_ids: ["abc-123", "def-456"]
date: "2026-05-16"
outcome: "verified_success"
repos: [employment-hero, rostering-api-service]
tags: [auto-logged]
---

Summarization Prompt

  • What was worked on — repo, ticket, brief description
  • Technical decisions — non-obvious choices + reasoning
  • Gotchas / surprises — bugs discovered, edge cases
  • Commands / patterns — key bash, code patterns
  • Pending tasks — unresolved items

Key Design Decisions

  • utility_score NOT stored — computed at query time as BM25
  • Same-day files append rather than overwrite
  • Transcript truncated to ~8,000 chars for LLM context
  • Output uses [[wikilinks]] for cross-referencing
05

Automation Schedule

Three cronjobs keep the loop running

Crontab

TimeScriptPurpose
30 23 * * 6 sync_sessions.py Saturday 23:30 — Process Claude sessions from last 7 days. Heuristic filter → LLM classify → aggregate daily notes → archive trash
0 0 * * 0 ingest.py --incremental Sunday 00:00 — After sync finishes. Index only files modified since last run. New session notes get picked up automatically.
0 1 1 * * ingest.py --full 1st of month 01:00 — Wipe ChromaDB collection and rebuild from scratch. Prevents index drift and stale embeddings.

📋 The full re-index runs at 01:00 to avoid racing with the weekly incremental at midnight.
Both nomic-embed-text and llama3:8b are already pulled via Ollama.

Repo Layout

Repository

  • ingest.py — Obsidian → ChromaDB indexer
  • server.py — FastMCP server (query + context tools)
  • sync_sessions.py — Session sync pipeline
  • chroma_db/ — Persistent vector store (gitignored)
  • venv/ — Python virtual environment

Obsidian Vault

  • Projects/ — Active engineering notes + _Brief.md
  • Knowledge/ — Evergreen technical notes
  • Logs/ — Auto-generated daily session notes
  • System/Staging/ — RESEARCH tier (future)
  • Daily/ — Manual daily logs (excluded from ingest)
